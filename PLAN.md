# План: Починка дашборда Яндекс.Директ

## Контекст

Дашборд (`dashboard.html`) — single‑file SPA на Chart.js + SheetJS для визуализации Excel-выгрузок Яндекс.Директа. Проблема: при загрузке файла в новом формате «Мастер отчётов» (июнь 2026) дашборд ничего не показывает — ни ошибок, ни данных.

**Корневая причина:** словарь `COL_META` заточен под старый формат колонок (май 2026). Новый формат использует другие названия: `Название кампании` вместо `Кампания`, `Расход, ₽` вместо `Расход (руб.)`, `CPC, ₽` вместо `Ср. цена клика (руб.)` и т.д. Неизвестные колонки попадают в авто-детект с неверным форматом — деньги не суммируются, KPI пустые, графики не строятся.

**Объём данных:** ~2 000 строк, 2 кампании, детализация по поисковым запросам.

## Подход

Минимальные изменения в `dashboard.html`:
1. Расширить `COL_META` новыми названиями колонок из «Мастер отчётов»
2. Улучшить fallback-детекцию (находить `₽`, ключевые слова `расход`/`cpa` для sum-агрегации)
3. Добавить защиту от тихих отказов (try/catch, сообщение при пустых метриках)
4. Пофиксить мелкие баги по пути (защита `sortCol`, сброс сортировки)

Старый формат (май) не должен сломаться.

## Файлы для изменения

- `dashboard.html` (единственный файл)

## Реиспользование

- `COL_META` — существующий словарь, расширяем, не переписываем
- Существующие fallback-функции (`getMeta`, `detectColumns`) — улучшаем, не заменяем
- Паттерн обработки ошибок: в проекте нет try/catch, добавляем по месту
